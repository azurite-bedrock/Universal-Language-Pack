# Contributing to the Universal Language Pack

Thank you for your interest in contributing! This project welcomes contributions from translators, developers, and community members.

This document covers the contribution process and the licensing terms that apply to contributions made on or after **version 0.4.0**.

## Where to Contribute

There are two main places to contribute:

- **Translations** - Submitted via our Crowdin project. This is the right place for adding, correcting, or improving translations and language data. _(See the project README for the current Crowdin link.)_
- **Code, scripts, and tooling** - Submitted via pull requests on GitHub. This includes the scripts that generate the resource pack, build tooling, automation, CI configuration, and documentation.

If you're not sure where something belongs, open an issue and ask first.

## How to Contribute

### Translations

1. Sign up on Crowdin and join the project (link in the README)
2. Choose your target language, or request a new one if it isn't listed
3. Translate strings, following the style notes pinned in the Crowdin project
4. Submitted translations are reviewed by language proofreaders before being merged into the next release

If you'd like to become a proofreader for a language, reach out via Crowdin, Discord or open an issue on GitHub.

### Code and Scripts

1. **Open an issue first** for non-trivial changes, so we can align on the approach before you spend time on it
2. Fork the repository
3. Create a feature branch (`feat/short-description` or `fix/short-description`)
4. Make your changes; follow existing code style
5. Sign off your commits with `-s`
6. Open a pull request describing what you changed and why

For small fixes (typos, obvious bugs, doc tweaks) you can skip step 1 and just open the PR.

### Documentation

Documentation lives in the repository as Markdown. Documentation contributions follow the same process as code contributions (PR, sign-off) but are licensed as content under CC BY-NC-SA 4.0.

## Code of Conduct

Be respectful, be patient, and assume good faith. The Project serves a wide international community of players and translators, many of whom are contributing in a language other than their native one.
Discriminatory, harassing, or hostile behavior will not be tolerated and may result in removal from the project's spaces.

## Questions

If you're unsure about anything, licensing, scope, style, process, open an issue or ask in our community channels before doing significant work. We'd much rather answer a question up front than ask you to redo work after the fact.
